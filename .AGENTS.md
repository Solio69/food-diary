# Конфигурация агентов проекта

Этот vault используется как личная система дневника питания: отчеты по приемам пищи, база продуктов, рецепты, фото еды и будущие расчеты калорий и КБЖУ.

Агент помогает со структурой данных, оформлением Markdown-документов, поддержкой базы знаний и подготовкой фактических отчетов. Агент не является консультантом по питанию, не ставит диагнозы, не дает диетологических рекомендаций и не делает выводы о стратегии питания.

## Рабочая среда

- Корень проекта: `D:\anastasiaStorage\Дневник питания`.
- Пользователь работает с файлами в Obsidian.
- Все служебные файлы и папки для агентов должны начинаться с точки, чтобы быть скрытыми в Obsidian.
- Основной язык документов: русский.
- Предпочтительная кодировка Markdown-файлов: UTF-8.

## Текущая структура

- `.agents/` - служебная папка агентов.
- `.agents/skills/` - локальные процедурные skills.
- `.agents/skills/nutrition-report.md` - skill для составления фактического отчета по питанию.
- `.GIT_POLICY.md` - обязательная политика работы AI-агентов с Git.
- `Отчеты/` - пользовательская папка для отчетов и будущих расчетов.
- `Продукты/` - база продуктов.
- `Рецепты/` - база рецептов.
- `Фото/` - фото еды, продуктов, упаковок и этикеток.

## Git-политика

Перед любыми действиями, связанными с Git, агент обязан прочитать и соблюдать `.GIT_POLICY.md`. Этот файл является единственным источником правил Git для проекта.

## Правила работы с данными питания

- Не придумывать граммовки, калории, БЖУ, состав, время приема пищи или факты о продукте.
- Если данных не хватает, явно писать `не указано`.
- Разделять фактический слой, расчеты и анализ.
- Фактический отчет фиксирует только источники, продукты, порции, фото, активность и wellness-практики.
- Расчет калорий и КБЖУ должен быть отдельным документом или отдельным будущим агентным процессом.

## Skill отчетов

Для задач вида "составить отчет по питанию" использовать `.agents/skills/nutrition-report.md`.

Ключевые правила skill v1:

- Создавать или обновлять файл `Отчет по питанию YYYY-MM-DD.md`.
- Не добавлять H1 с названием файла.
- Структура отчета: `## Источники`, затем H2-блоки приемов пищи в хронологическом порядке, затем `## Активность и wellness-практики`.
- Внутри приема пищи использовать только `### Продукты` и `### Фото`, пока пользователь не утвердит другую структуру.
- Фото оформлять HTML-сеткой с относительными путями.
- Целевая папка для фото: `Фото/`; на входе фото могут временно лежать в корне vault.
- Если фото не переданы, оставлять блок фото пустым или писать `- Фото не переданы.`
- Если фото есть, но прием пищи не распознан, добавлять `## Фото без привязки`.

Внимание: текущий skill v1 указывает создавать отчет в корне vault. Если пользователь отдельно попросит изменить место хранения отчетов, сначала обновить соответствующий skill или явно зафиксировать новое правило.

## Стиль файлов

- Писать документы коротко, структурно и пригодно для Obsidian.
- Использовать wiki-ссылки Obsidian там, где это удобно для связей между продуктами, рецептами, отчетами и источниками.
- Не добавлять служебный агентный шум в пользовательские документы.
- Не создавать видимые служебные файлы без прямой необходимости.
- В описаниях процессов, конфигурациях и skills фиксировать только актуальное состояние работы. Не описывать прежние варианты, переходы между версиями и историю изменений; неактуальную информацию удалять или заменять актуальной.
- Сохранять пользовательские правки и не откатывать изменения, сделанные не агентом.
