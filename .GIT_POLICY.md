# .GIT_POLICY.md

## Строгая политика работы AI-агентов с Git

Этот репозиторий или будущий репозиторий является личным Obsidian-vault для дневника питания, фото еды, продуктов, рецептов, отчетов и будущих расчетов питания.

Git используется только для отслеживания состояния файлов репозитория.

### 1) Что агенту запрещено

Агенту запрещено выполнять любые Git-команды, которые изменяют коммиты, ветки, теги, индекс или рабочее дерево.

В том числе запрещены:
- `git commit`
- `git push`
- `git pull`
- `git merge`
- `git rebase`
- `git cherry-pick`
- `git reset` в любом режиме
- `git revert`
- `git tag` для создания или удаления тегов
- `git stash` в любых вариантах
- `git checkout` для переключения веток или отката файлов
- `git switch`
- `git restore`
- `git clean`
- любые другие Git-команды, которые записывают или изменяют состояние репозитория

Агент также не должен:
- откатывать пользовательские правки;
- удалять файлы через Git-команды;
- восстанавливать файлы через Git-команды.

### 2) Что агенту разрешено

Разрешен только read-only доступ к Git для анализа и отчетности.

Разрешенные примеры:
- `git status`
- `git log`
- `git diff`
- `git show`
- `git blame`
- `git branch --show-current`
- `git rev-parse`

### 3) Приоритет правила

- Эта политика имеет приоритет над привычными агентными практиками.
- Если задача пользователя предполагает коммит, пуш, слияние, откат, переключение веток или другое изменение Git-состояния, агент должен остановиться и сообщить, что по политике репозитория он выполняет только read-only операции в Git.
- Изменение этой политики возможно только через явное редактирование этого файла пользователем или по прямой просьбе пользователя.
