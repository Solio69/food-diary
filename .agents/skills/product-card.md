# Skill: карточка продукта

Использовать, когда в материалах питания встречается новый продукт и для него нужно создать или обновить карточку в `Продукты/`.

## Роль skill

Создать фактическую карточку продукта для базы знаний Obsidian. Карточка продукта хранит данные из описания автора, фото упаковки, этикетки или другого явного источника.

Не рассчитывать фактические калории и БЖУ приема пищи или дня.

## Что считать продуктом

Продукт - отдельная покупная или базовая пищевая единица: упаковка, банка, напиток, крупа, хлебцы, йогурт, соус, овощ, фрукт и похожие сущности.

## Имя файла

Создавать или обновлять файл:

```md
Продукты/Название продукта.md
```

Название файла должно быть понятным для поиска в Obsidian. Очевидные опечатки в названии исправлять до создания файла. Если бренд важен для различения продуктов, включать бренд в название файла.

## Шаблон файла

Каждая карточка продукта должна создаваться по шаблону из `.templates/Продукт.md`. При обновлении существующей карточки сохранять структуру шаблона.

Поле `photo` - чекбокс наличия фото. Если к карточке добавлены фото продукта, упаковки или этикетки, агент ставит `photo: true`.

Поле `done` - пользовательский чекбокс проверки карточки. Агент никогда не меняет `done`.

Если автор подтверждает карточку со справочными данными через `done`, карточка считается валидной для базы.

Сами фото крепить ниже в отдельном блоке `### Фото` через Obsidian image-embeds с шириной `260`. HTML `img` в карточках продуктов не использовать.

## Структура карточки

```md
---
topik:
  - Дневник питания
tags:
  - Продукт
photo: true
done:
---
### Описание

- Название: **...**.
- Бренд: **...**.
- Вид продукта: ...
- Масса/объем упаковки: ...

### Данные с упаковки

- Состав: ...
- Калорийность на 100 г/100 мл: ...
- Белки на 100 г/100 мл: ...
- Жиры на 100 г/100 мл: ...
- Углеводы на 100 г/100 мл: ...
- Пищевая ценность на порцию: ...

### Фото

![[Фото/photo-file.jpg|260]]

### Использовано в блюдах

- [[Название блюда]] — ...

### Использование в питании

- [[Отчет по питанию YYYY-MM-DD]] — ...
```

Если читаемых данных с упаковки или этикетки нет, вместо `### Данные с упаковки` использовать:

```md
### Справочные данные

- Калорийность на 100 г/100 мл: ...
- Белки на 100 г/100 мл: ...
- Жиры на 100 г/100 мл: ...
- Углеводы на 100 г/100 мл: ...
- Основание: справочные данные, не с упаковки.
```

## Правила заполнения

- Блок `Источники` в карточках продуктов не использовать.
- Блоки карточки продукта оформлять H3-заголовками.
- После `### Описание` использовать ровно один блок пищевых данных: `### Данные с упаковки`, если данные с упаковки читаются на фото или явно указаны автором; `### Справочные данные`, если данных с упаковки нет или они не читаются.
- В блоке пищевых данных обязательно фиксировать калорийность и белки на 100 г/100 мл; жиры и углеводы фиксировать, если есть данные.
- Если данные с упаковки читаются частично и в них нет обязательной калорийности или белков, недостающее обязательное значение можно добавить в `### Данные с упаковки` как справочное с явной пометкой `справочно, не с упаковки`.
- Порядок блоков: `### Описание`, затем `### Данные с упаковки` или `### Справочные данные`, затем `### Фото`, `### Использовано в блюдах`, `### Использование в питании`.
- В блоке `### Описание` выделять жирным значения полей `Название` и `Бренд`, если бренд указан.
- Если бренд не указан, писать `- Бренд: не указано.` без жирного выделения.
- Первая запись в `### Использование в питании` фиксирует контекст, из которого появилась карточка продукта.
- В блоке `### Использовано в блюдах` фиксировать блюда/рецепты, где продукт используется как ингредиент.
- В блоке `### Использование в питании` фиксировать прямое использование продукта в отчетах по питанию.
- Фактические данные продукта переносить только из описания автора или читаемых фото.
- Читаемые данные с этикетки фиксировать как фактическую информацию продукта.
- Если данных с упаковки нет или они не читаются, добавлять блок `### Справочные данные` вместо `### Данные с упаковки`.
- В блоке `### Справочные данные` указывать `Основание: справочные данные, не с упаковки`.
- Фото продукта, упаковки и этикетки фиксировать в карточке продукта через `![[Фото/...|260]]` и не добавлять в отчет по питанию.
- Не добавлять пути к фото в YAML-поле `photo`.
- Не менять YAML-поле `done`.
- Если фактическое значение на упаковке не читается и не относится к обязательным справочным пищевым данным, писать `не указано`.
- Если бренд не указан или не читается, писать `не указано`.
- Если продукт уже есть в `Продукты/`, обновлять существующую карточку только фактическими данными из текущих материалов.
- Использовать Obsidian-ссылки на отчеты, рецепты и связанные продукты там, где это помогает связать базу знаний.
