# Skill: отчет по питанию и

Локальный процедурный skill для агента питания. Использовать, когда автор отдельно передает описание приемов пищи и фото еды, упаковок или этикеток, а агент должен создать файл `Отчет по питанию YYYY-MM-DD.md` в корне папки.


## Роль skill

Собрать структурированный источник данных для nutrition-agent. Это фактический слой, а не расчет и не анализ.

Не анализировать рацион, не считать калории и БЖУ, не давать рекомендации по диете, не делать выводы о стратегии питания. Подсчет калорий должен выполняться отдельно nutrition-agent в будущем файле расчета, например `Расчет питания YYYY-MM-DD.md`.

## Входные материалы

- Описание приемов пищи в формате, который автор передаст отдельно. Формат пока не зафиксирован: это может быть текст, таблица, файл, экспорт из другого приложения или другой рабочий формат.
- Отдельные фото еды/продуктов/этикеток - основной источник изображений для отчета по питанию, если автор добавила их в корень или указанную папку.

## Имя файла

Создавать или обновлять файл:

```md
Отчет по питанию YYYY-MM-DD.md
```

Не добавлять отдельный H1 с названием файла: в Obsidian имя файла уже работает как заголовок.

## Структура отчета

Структура v1 состоит только из трех частей:

1. `## Источники`
2. Блоки приемов пищи в хронологическом порядке.
3. `## Активность и wellness-практики`

```md
## Источники

- Описание питания: [[...]]

## Название приема пищи — время приема не указано

### Продукты

- Продукт — размер/граммовка.

### Фото

<div style="display:grid; grid-template-columns:repeat(4,minmax(0,1fr)); gap:6px; max-width:560px;">
  <img src="photo-1.jpg" style="width:100%;">
  <img src="photo-2.jpg" style="width:100%;">
  <img src="photo-3.jpg" style="width:100%;">
  <img src="photo-4.jpg" style="width:100%;">
</div>

## Активность и wellness-практики

- Сон: ...
- Активность: ...
- Практики: ...
```

## Правила заполнения

- Блок `## Источники` держать в начале.
- Приемы пищи располагать в хронологическом порядке.
- Каждый прием пищи оформлять отдельным H2-блоком.
- В каждом приеме пищи использовать только подблоки `### Продукты` и `### Фото`, пока автор не утвердит новую структуру.
- Если в тексте есть реальное время приема пищи, использовать его.
- Если время приема пищи неизвестно, писать: `время приема не указано`.
- Внутри приема пищи указывать только продукты/напитки, размеры порций, граммовки и относящиеся фото.
- Не придумывать граммовки, калории, БЖУ, состав или время приема пищи.
- Если продукт виден на фото, но не указан в описании питания, можно добавить его с пометкой `добавлено по фото`.
- Если размер не известен, писать `размер не указан`.
- Вопросы, замечания и рекомендации агента писать в отдельный рабочий разбор питания в корне папки. Не писать их в отчет по питанию.
- Контентные идеи не писать в отчет по питанию.
- Не добавлять таблицы калорий, дневные итоги, оценку дефицита/профицита или рекомендации. Это зона будущего расчета питания.

## Фото

Фотографии оформлять HTML-сеткой, потому что:

- Markdown-таблицы ломаются из-за `|250` внутри Obsidian-embed.
- Inline-embeds переносятся непредсказуемо.
- HTML-сетка дает стабильный компактный коллаж.

Правило v1:

```md
<div style="display:grid; grid-template-columns:repeat(4,minmax(0,1fr)); gap:6px; max-width:560px;">
  <img src="photo-file-1.jpg" style="width:100%;">
  <img src="photo-file-2.jpg" style="width:100%;">
</div>
```

Использовать путь к фото относительно файла отчета. Если отчет и фото лежат рядом, писать только имя файла. Если фото лежит в `_Фотки еды/`, использовать относительный путь от отчета, например `_Фотки еды/photo-file.jpg`. Агент может переносить и переименовывать фото внутри `Дневник питания/` и `_Фотки еды/`, если это нужно для понятного отчета.

Если фото для приема пищи не переданы, оставить блок `### Фото` пустым или написать:

```md
- Фото не переданы.
```

Если фото есть, но прием пищи не распознан, добавить в конец отчета блок:

```md
## Фото без привязки

<div style="display:grid; grid-template-columns:repeat(4,minmax(0,1fr)); gap:6px; max-width:560px;">
  <img src="photo-file.jpg" style="width:100%;">
</div>
```

И добавить вопрос или замечание в отдельный рабочий разбор питания в корне папки.
